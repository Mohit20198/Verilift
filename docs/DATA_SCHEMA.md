# VeriLift Data Schema

## File 1: data/campaigns.csv

| Column | Type | Required | Description |
|---|---|---:|---|
| campaign_id | string | yes | Unique campaign identifier |
| campaign_name | string | yes | Campaign display name |
| network_name | string | yes | RetailNet A, RetailNet B, or RetailNet C |
| campaign_invoice | float | yes | Total invoice amount in INR |
| claimed_lift | float | yes | Network-reported relative lift as decimal, e.g. 0.20 |
| claimed_revenue | float | yes | Revenue claimed by network |
| base_payment_pct | float | yes | Base guaranteed payment percentage as decimal |
| performance_pool_pct | float | yes | Performance-linked pool percentage as decimal |
| start_date | date | yes | Campaign start date |
| end_date | date | yes | Campaign end date |

## File 2: data/test_control.csv

| Column | Type | Required | Description |
|---|---|---:|---|
| campaign_id | string | yes | Foreign key to campaigns.csv |
| group_type | string | yes | `exposed` or `holdout` |
| users | int | yes | Number of users in group |
| conversions | int | yes | Number of converting users |
| revenue | float | yes | Revenue generated by group |
| new_customers | int | yes | New customers in group |
| repeat_customers | int | yes | Repeat customers in group |

Each campaign must have exactly two rows in this table:
- one exposed row
- one holdout row

## File 3: data/network_history.csv

**Note:** `network_history.csv` represents past campaigns used for historical reliability scoring. `campaigns.csv` and `test_control.csv` represent current demo campaigns. Phase 4 should calculate reliability using both historical campaign data and current verified results if needed.

| Column | Type | Required | Description |
|---|---|---:|---|
| network_name | string | yes | Retail media network |
| campaign_id | string | yes | Campaign identifier |
| claimed_lift | float | yes | Network claimed lift |
| verified_lift | float | yes | VeriLift verified lift |
| deviation | float | yes | Claimed vs verified deviation |
| reliability_score | float | yes | Network trust score |

## Synthetic Data Story

**Core Rule:** Synthetic data must not be purely random. The generated data must support the demo story mathematically. Claimed lift, claimed revenue, exposed conversions, holdout conversions, and verified lift must mathematically align. The numbers should look realistic and consistent.

RetailNet A:
- Claimed lift around 18–22%
- Verified lift around 9–12%
- Moderate overstatement

RetailNet B:
- Claimed lift around 12–15%
- Verified lift around 11–14%
- Reliable

RetailNet C:
- Claimed lift around 22–26%
- Verified lift around 5–8%
- Heavy overstatement
